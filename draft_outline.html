Lesson 2: Speaking to the Computer - Your First Python Program

Objective: Write your first Python code and understand print(), comments, strings, and numbers.

What is a Programming Language? (10 mins)

Think of a programming language like a recipe for a computer.

It needs very specific, step-by-step instructions in a language it understands. We will use Python, which is famous for being powerful and easy to read.

"Hello, Hong Kong!" (15 mins)

Let's give the computer its first instruction. The print() command tells the computer to display something on the screen.

# This is a comment. The computer ignores it. It's for humans!
print("Hello, World!")

# Let's change the message to something local.
print("Hello, Hong Kong!")

The text inside the quotation marks " " is called a string.

Data Types: Text and Numbers (15 mins)

Python knows the difference between text (strings) and numbers (integers).

# This is a string (text)
print("This is a head of lettuce")

# This is an integer (a whole number)
print(100)

# We can also do math with numbers!
print(5 + 3) # Output will be 8

Lesson 2 Recap

Key terms we learned today:

  • print(): The command to display output.
  • # Comment: A note for humans that the code ignores.
  • String: Text data, always inside quotes.
  • Integer: Whole number data, without quotes.

Lesson 3: Making Decisions - Variables and 'If' Statements

Objective: Understand variables and use if-elif-else to make decisions in code.

Variables as Labeled Boxes (15 mins)

A variable is like a labeled box where you can store information. We give it a name and put data inside it using the = sign.

# Storing a string in a variable
plant_type = "Lettuce"

# Storing an integer
days_grown = 25

# Storing a True/False value (a Boolean)
is_healthy = True

# We can then print the variables to see what's inside
print(plant_type)

Making Choices with 'if' (25 mins)

The if statement lets our program make decisions. It checks if a condition is true.

water_level_cm = 5

if water_level_cm < 10:
    print("ALERT: Water level is low! Turn on the pump.")
else:
    print("Water level is OK.")

This is the core logic of our AI! It will check the plant's health and decide what to do.

Project Link: Deciding the Box Color

Here's a simplified version of the logic from our final project. We use if, elif (else if), and else.

health_status = "spoiled"

if health_status == "fresh":
    print("Draw a GREEN box.")
elif health_status == "fair":
    print("Draw a YELLOW box.")
else:
    print("Draw a RED box.")

This simple choice is how the program decides what color box to draw around a plant!

Lesson 4: The Programmer's Toolkit

The Programmer's Tools

  • Visual Studio Code (VS Code): A powerful, free text editor for writing code. It has a built-in terminal, which is our command line.

Lesson 7: A Closer Look - From Detection to Decision

Objective: Examine specific code snippets to see how a decision is turned into a visual output.

Code Deep Dive 1: The Color Map

How does the program know which color to use for the box? It uses a Python dictionary.

In backend.py, we find this:

self.COLOR_MAP = {
    'fresh': (0, 255, 0),   # Green in (B, G, R) format for a healthy plant
    'spoiled': (0, 0, 255)  # Red in (B, G, R) format for a sick plant
}

This links a text "key" (like 'fresh') to a "value" (the color code for green). It's a simple lookup table.

Code Deep Dive 2: Drawing the Box

In realtime_detection.py, we find the code that actually draws the rectangle.

# 1. Look up the color based on the health status result
display_color = self.backend.COLOR_MAP.get(health_status, (255, 255, 255))

# 2. Use the OpenCV library (cv2) to draw the rectangle
cv2.rectangle(display_frame, (x1, y1), (x2, y2), display_color, 2)

This shows how we connect the backend's decision ('fresh') to a specific color from the map, and then use that color to draw on the screen. It's all connected!

Troubleshooting: A Learning Opportunity! (15 mins)

Things don't always work on the first try, and that's okay! It's part of programming.

  • Problem: "Camera not found" or black screen.
    Solution: Check if the webcam is plugged in. You might need to change CAMERA_ID = 0 to 1 in the code.
  • Problem: "ModuleNotFoundError: No module named 'cv2'".
    Solution: You forgot to run pip install -r requirements.txt or your virtual environment isn't active.
  • Problem: "File not found: ... .tflite".
    Solution: Make sure the model file is in the same folder as your Python scripts.

Lesson 9: Calibrating and Measuring Growth

Objective: Understand calibration and how the system can track plant growth over time.

What is Calibration? (15 mins)

Right now, the computer sees the world in pixels, not centimeters.

Calibration is how we teach the computer the relationship between pixels and a real-world measurement. We use a reference object of a known size (e.g., a 10cm x 10cm white card).

When you run the program, you can press c with the card in view. The program measures the card in pixels, does the math, and then knows how to convert any pixel measurement to centimeters!

Calibrated: 45.7 px/cm

Applications in Aquaponics: Growth Monitoring

This tool is perfect for monitoring plant growth in an aquaponics or hydroponics system.

By pressing g, the system enters growth mode. It logs the plant's size (area, perimeter) into a growth_data.csv file, creating a data record over time.

timestamp,class_name,area_cm2,perimeter_cm
2023-10-27 20:01:00,Lettuce,55.2,28.1
2023-10-28 20:01:00,Lettuce,58.9,29.5
2023-10-29 20:01:00,Lettuce,62.1,30.7

This allows us to track growth rates, know when to harvest, and see if a plant is not growing well.

IoT Alerts with MQTT (15 mins)

MQTT is a lightweight messaging system for Internet of Things (IoT) devices.

Analogy: It's like a digital noticeboard for our farm.

  • Our program is a Publisher. When it detects a sick plant, it posts a message to a topic like: aquaponics/alerts.
  • The greenhouse manager's phone or a central dashboard is a Subscriber. It gets the alert instantly, allowing for a fast response to save the crop!

This is how we build a truly smart and automated farming system.

Lesson 11: Your First Code Modification

Objective: Make a simple, safe modification to the code to change the system's behavior.

Challenge: Change the Alert Color!

The standard red box for a sick plant is good, but we want a more noticeable purple box.

Your task is to find the right line of code and change it.

Hint: Which file controls the colors? Where did we see the color codes before?

Guided Modification (25 mins)

1. Open backend.py.

2. Find the COLOR_MAP dictionary.

3. This is the original line for a spoiled/sick plant (B, G, R format):

'spoiled': (0, 0, 255), # Red

4. The RGB code for purple is (128, 0, 128). In BGR format, that's (128, 0, 128).

5. Change the line to:

'spoiled': (128, 0, 128), # Purple

6. Save the file and re-run python realtime_detection.py. The box for a sick plant should now be purple. Congratulations!

AI Ethics & Limitations (10 mins)

AI is powerful, but not perfect. We must think critically.

  • Bias in Data: What if we only trained our AI on one type of lettuce? It might not recognize other types. The AI is only as good as the data we give it.
  • Accuracy vs. Trust: What if the AI says a plant is healthy, but it has a disease on the underside of a leaf? We must use AI as a tool to assist our judgment, not replace it.
